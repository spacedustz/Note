## 🧑🏻‍💻 Version APIs for Admin
---

## 💡 요구사항

개발할 기능은 총 두가지이다.
어드민 유저만 접근 가능한 API 두개를 생성한다.
<br>
이번 과제에서의 상태는 version entity의 status Attribute로써
true로 되어있을때는 강제업데이트를 진행하지 않고
false로 되어있을때는 강제업데이트를 진행하는 Attribute이다.
<br>
Patch API의 시간복잡도를 고려해 볼 것
Get API는 Reverse된 순서로 전달해 볼 것 (최신버전이 앞으로 오도록 반환)
<br>
### 주의 사항
어드민 유저만 접근 가능해야한다
<br>
### 내용

- 현재 버전 전체 상태를 리스트로 반환하는 Get API
```
  List< DTO> 
  필요반환정보 : {
  String id,
  String 버전,
  String os,
  Boolean 상태
  }
```
 <br>
- 버전 상태를 바꿀 수 있는 Patch API
  입력 값 List<Long, boolean>
  각각 id의 값을 boolean 값으로 status 교체
```json
  [

  ​	{id : 215, status : false},
  ​	{id : 230, status : true}

  ]
```

  성공시 200 / 일부 성공시에도 200 (단 일부 성공시 get api와 같은 내용 반환할 것)
  실패시 에러코드는 자유 (상황에 맞는 에러코드 404 등으로 할당)

- 소팅은 최대한 배제

### HealthCheckService

```kotlin
/**  
 * @author 신건우  
 * @desc 서버 버전 리스트 역순 조회 기능  
 */  
fun getVersionsDesc() : List<ServerInfoDTO.Admin> {  
  
    val versions = try {  
        versionRepository.searchAll().reversed()  
    } catch (e : Exception) {  
        log.error("SERVER-001 : 서버 버전 리스트 불러오기 실패 - $e")  
        throw CommonException("SERVER-001", HttpStatus.NOT_FOUND)  
    }  
  
    log.info("서버 버전 리스트 역순 조회 성공 - [${versions.size}]")  
  
    return versions.stream().map { ServerInfoDTO.Admin.ModelMapper.entityToDto(it) }.toList()  
}

/**  
 * @author 신건우  
 * @param data : 버전 번호와 Status 리스트 정보  
 * @desc 서버 버전 상태 일괄 변경 기능  
 */  
@Transactional  
fun patchStatus(  
    data : List<ServerInfoDTO.VersionUpdate>  
) : List<ServerInfoDTO.Admin> {  
  
    log.info(data)  
  
    var ids = data.stream().map { it.id }.toList()  
  
    var versions = try {  
        versionRepository.findAllById(ids)  
    } catch (e : Exception) {  
        log.error("SERVER-001 : 버전 검색 실패 - $e")  
        throw CommonException("SERVER-001", HttpStatus.NOT_FOUND)  
    }  
  
    if (ids.size != versions.size) {  
        log.error("SERVER-004 : [데이터 오염] 잘못된 id 번호가 입력됨")  
        throw CommonException("SERVER-004", HttpStatus.BAD_REQUEST)  
    }  
  
    for (index : Int in 0 until versions.size) {  
  
        if (data[index].id != versions[index].id) {  
            log.error("SERVER-001 : 검색 실패 - [ 입력된 버전값 : ${data[index].id} ]")  
            throw CommonException("SERVER-001", HttpStatus.BAD_REQUEST)  
        }  
  
        versions[index].modifyingVersionStatus(data[index].status)  
    }  
  
    log.info("서버 버전 일괄 변경 성공")  
  
    return versions.stream().map { ServerInfoDTO.Admin.ModelMapper.entityToDto(it) }.toList()  
}
```

### HealthCheckController

```kotlin
/**  
 * @author 신건우  
 * @desc 서버 버전 리스트 역순 조회 API */@Auth(Auth.Role.ADMIN)  
@GetMapping("/versions")  
fun getVersionsDesc(): ResponseEntity<List<ServerInfoDTO.Admin>> {  
    return ResponseEntity(healthCheckService.getVersionsDesc(), HttpStatus.OK)  
}  
  
/**  
 * @author 신건우  
 * @param data : 바꿀 버전의 DTO 입력 데이터  
 * @desc 서버 버전 일괄 변경 API * */@Auth(Auth.Role.ADMIN)  
@PatchMapping("/patch-stat")  
fun patchStatus(  
    @RequestBody data : List<ServerInfoDTO.VersionUpdate>  
) : ResponseEntity<List<ServerInfoDTO.Admin>> {  
    return ResponseEntity(healthCheckService.patchStatus(data), HttpStatus.OK)  
}
```

### ServerInfoDTO
```kotlin
class ServerInfoDTO {  
  
    data class Update (  
        var data : List<String> = listOf()  
    )  
  
    data class VersionUpdate (  
        var id : Long,  
        var status : Boolean  
    )  
  
    data class Admin (  
        var id : Long? = null,  
        var version : String = "",  
        var os : String? = "",  
        val status : Boolean = true  
    ) {  
        object ModelMapper {  
            fun entityToDto(form : Version) : Admin {  
                return Admin(  
                    id = form.id,  
                    version = form.version,  
                    os = form.os,  
                    status = form.status  
                )  
            }  
        }  
    }  
}
```
