![image](https://s3.ap-northeast-2.amazonaws.com/urclass-images/vMWQFkj_05d-Vk52EASLt-1648425920501.png) 로그인 창은 웹페이지마다 조금씩 차이는 있지만, 전체적인 구조는 거의 비슷합니다. 잠시 네이버의 로그인 페이지를 보면서 어떤 요소들이 있는지 잠시 생각해 보겠습니다.

먼저, 아이디 또는 이메일 주소를 입력할 입력 폼이 필요합니다. 또, 비밀번호를 입력할 수 있는 입력 폼도 필요하겠네요. 또, 로그인 버튼도 꼭 필요합니다. 이 정도만 있어도 기본적인 로그인 페이지의 구조가 됩니다. 여기에 로그인 상태를 유지할 수 있도록 체크박스를 하나 만들고, 회원가입 페이지로 이동할 수 있는 링크 하나를 넣으면 조금 더 완성도 있는 로그인 페이지가 될 것입니다.

여러분이 개념 학습에서 다룬 내용을 잘 학습했다면, 이 요소를 만들기 위해 각각 어떤 태그를 사용해야 할지 어렵지 않게 떠오르실 겁니다. 이번 실습에서는 VScode를 이용해서 직접 HTML 태그를 작성해서 간단한 로그인 페이지를 만들어 보도록 하겠습니다. JavaScript나 CSS를 사용하지 않기 때문에 실제로 로그인 기능이 작동하거나, 디자인이 추가된 로그인 페이지를 만들 수는 없지만, 로그인 페이지에 주로 사용되는 HTML 태그를 직접 사용하고 브라우저에서 확인하는 것은

이번 실습에서는 Chapter1-2에서 만든 login.html 파일을 사용합니다.

### 1. 아이디, 비밀번호 입력 창 만들기

아이디를 입력하는 입력 폼을 만들기 위해서는 어떤 태그를 사용해야 할까요? input 태그를 사용하면 됩니다.

우선 login.html파일에 아래 코드를 붙여 넣기 하세요.

```
1
```

위 코드를 입력했다면 브라우저가 여러분이 입력한 input태 그를 읽어서 입력 폼을 보여줍니다. 어떻게 확인할 수 있을까요? 왼쪽 사이드바에서 파일명을 오른쪽 클릭해 보면 나오는 메뉴 중 가장 하단에 Open In Default Browser를 클릭하면 됩니다. (이 메뉴는 VScode를 설치할 때 함께 설치한 Extension인 'open in browser'를 설치했을 경우 나타나는 메뉴입니다.) ![image](https://s3.ap-northeast-2.amazonaws.com/urclass-images/Zml-IX6gDas_gYPtdGYqF-1648425978932.png)

브라우저에서 새 창이 열리면서 HTML 파일이 열립니다. 브라우저가 여러분이 입력한 input 태그를 읽어서 입력 폼을 생성한 것을 확인할 수 있습니다. 현재는 여러분의 로컬 컴퓨터에 있는 HTML 파일이 열린 것이기 때문에 만약 여러분이 파일을 수정했다면, 새로고침을 해주어야 수정한 내용이 반영됩니다. ![image](https://s3.ap-northeast-2.amazonaws.com/urclass-images/Lgmn6njZ1u2ey3BEBoDH_-1648426623464.png)

아이디를 입력할 수 있는 폼을 만들었다면, 비밀번호를 입력할 폼도 만들어 보겠습니다. 간단하게 생각하면 input 태그를 한 번 더 사용해서 입력 폼을 하나 더 만들면 됩니다.

```
1
2
```

브라우저에서 새로고침을 해보면, 입력 폼이 2개가 된 것을 확인할 수 있습니다. 하지만 아직은 두 입력폼이 똑같습니다. 아이디 입력 폼에는 type 속성을 text로, 비밀번호 입력 폼에는 type 속성을 password로 추가하겠습니다.

```
1
2
```

placeholder 속성을 사용하면, 입력 폼 안에 작성해야 하는 내용에 대한 힌트를 제공할 수 있습니다.

```
1
2
```

codesandbox를 통해 결과물을 확인할 수 있습니다. ID와 password에 각각 입력해 보고 차이점을 확인하세요!

### 2. 로그인 버튼 만들기

로그인 버튼을 만들기 위해서는 `<button>`태그를 사용합니다. 여는 태그와 닫는 태그 사이에는 값(value)을 넣을 수 있습니다.

```
1
2
3
```

생성된 버튼은 클릭이 가능하지만 버튼은 작동하지 않습니다. 현재는 로그인 페이지에 사용되는 태그의 종류를 확인하는 것으로도 충분합니다.
